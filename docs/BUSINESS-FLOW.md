# MediSlim 完整业务流 — Medvi模式中国复刻

## Medvi业务流深度拆解

### 美国原版流程

```
用户旅程：
发现广告 → 落地页 → 健康评估 → 付费订阅 → 医生远程问诊 → 开处方 → 药房配药 → 快递到家 → 复购追踪

系统流程：
┌─────────────────────────────────────────────────────────────────┐
│                        Medvi (前端层)                           │
│  品牌网站 / 广告投放 / 用户评估 / 支付结算 / 客服 / 复购管理       │
├─────────────────────────────────────────────────────────────────┤
│                  OpenLoop Health (医疗层)                        │
│  执业医师网络 / 远程问诊 / 处方开具 / 医疗合规 / 实验室对接         │
├─────────────────────────────────────────────────────────────────┤
│                  CareValidate (履约层)                           │
│  合作药房网络 / 合成药品生产 / 处方审核 / 物流配送 / 库存管理       │
├─────────────────────────────────────────────────────────────────┤
│                  其他合作伙伴                                     │
│  LabCorp/Quest (化验) / FedEx/UPS (物流) / Stripe (支付)          │
└─────────────────────────────────────────────────────────────────┘
```

### 关键业务节点

| 节点 | Medvi做法 | 时间线 |
|------|----------|--------|
| 1. 用户评估 | 在线问卷（排除禁忌症） | 即时 |
| 2. 付费 | 月订阅$179起，28天自动续费 | 即时 |
| 3. 医生问诊 | OpenLoop医师远程审核 | 24-48h |
| 4. 开处方 | 合成GLP-1处方（1-5mg递增） | 问诊后即时 |
| 5. 配药 | 合作药房配制+质检 | 1-2天 |
| 6. 发货 | FedEx快递，密封包装，附带注射器等耗材 | Day 0-2 |
| 7. 复购提醒 | Day 21-28邮件/短信提醒续费 | 自动 |
| 8. 复购 | 用户确认→自动扣款→药房自动发货 | 自动 |
| 9. 实验室检查 | Month 3后必须做化验（LabCorp/Quest） | 定期 |
| 10. 剂量调整 | 医生根据随访数据调整 | 按需 |

## 中国版业务流设计

### 合作伙伴架构（中国对标）

| 角色 | Medvi (美国) | MediSlim (中国) | 对接方式 |
|------|-------------|----------------|---------|
| **医疗层** | OpenLoop Health | 微医/好大夫/京东健康 | API对接 |
| **药房层** | CareValidate药房网络 | 大参林/益丰/老百姓 | API+人工 |
| **化验层** | LabCorp/Quest | 美年大/爱康国宾 | 报告上传 |
| **物流层** | FedEx | 顺丰/京东物流 | API对接 |
| **支付层** | Stripe | 微信支付/支付宝 | 商户API |
| **流量层** | Google/Meta广告 | 微信/抖音/小红书 | 广告后台 |
| **客服层** | AI+人工 | 企业微信+AI | 企微API |

### 完整业务流

```
【阶段1：获客】
    微信/抖音/小红书广告
           ↓
    落地页 (landing.py :8091)
           ↓
    留资（手机号+姓名+需求）
           ↓
    → 进入线索池 (data/leads.json)

【阶段2：评估】
    健康顾问30分钟内联系
           ↓
    引导完成AI健康评估 (app.py :8090)
           ↓
    AI分析（禁忌症筛查+风险分级+方案推荐）
           ↓
    → 生成评估报告

【阶段3：付费】
    展示方案+首月价格
           ↓
    微信支付/支付宝付款
           ↓
    → 订单创建 (data/products.json)
    → 状态：paid

【阶段4：医师审核】
    订单推送至互联网医院API
           ↓
    执业医师远程审核（24h内）
           ↓
    医师开具电子处方
           ↓
    → 状态：prescribed

【阶段5：配药发货】
    处方推送至合作药房
           ↓
    药房配药+质检
           ↓
    顺丰/京东发货（保密包装）
           ↓
    推送物流信息给用户
           ↓
    → 状态：shipped

【阶段6：收货用药】
    用户签收
           ↓
    健康顾问指导用药
           ↓
    每周用药日记（小程序记录）
           ↓
    → 状态：in_use

【阶段7：随访复购】
    Day 21：系统自动推送续费提醒
           ↓
    Day 25：健康顾问跟进效果
           ↓
    Day 28：自动扣款续费
           ↓
    → 药房自动发货下个月剂量
    → 状态：renewed

【阶段8：长期管理】
    每月：体重/指标追踪
    每3月：化验检查（推荐）
    按需：剂量调整（医师审核）
    → 持续复购循环
```

### 状态机

```
[新线索] → [已联系] → [已评估] → [已付费] → [医师审核中]
    ↓                                           ↓
 [流失]                              [已开处方] → [配药中]
                                                    ↓
                                              [已发货] → [已签收]
                                                            ↓
                                                      [用药中] → [续费提醒]
                                                                    ↓
                                                              [已续费] → (循环)
```

### 核心API接口设计

```python
# 互联网医院对接接口
POST /api/ih/create_consultation   # 创建问诊
POST /api/ih/check_status          # 查询问诊状态
POST /api/ih/get_prescription      # 获取处方

# 药房对接接口
POST /api/pharmacy/submit_order    # 提交配药订单
POST /api/pharmacy/check_status    # 查询配药状态
POST /api/pharmacy/get_tracking    # 获取物流单号

# 支付对接接口
POST /api/pay/create               # 创建支付
POST /api/pay/callback             # 支付回调
POST /api/pay/refund               # 退款

# 用户端接口
POST /api/user/register            # 注册
POST /api/user/assessment          # AI评估
POST /api/user/order               # 下单
POST /api/user/refill              # 续费
GET  /api/user/tracking            # 物流查询
```
